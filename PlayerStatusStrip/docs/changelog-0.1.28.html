Player Status HUD - 0.1.28

Release date: 2026-05-07

Stability release verified in-game: status strip stays visible after leaving to the main menu and loading a world again. Includes hotkey registration guard and HUD lifecycle fixes from recent patches.

Highlights

  • Hotkey F8: RegisterHotKeyFirst runs only if playerstatusstrip_reloadlayout is not already in Input.HotKeys — prevents StartClientSide from crashing on the second world session.
  • HUD lifecycle: strip element is disposed on LeftWorld and recreated or reopened on LevelFinalize so the overlay matches a fresh game session.
  • Earlier features retained: server boot creates playerstatusstrip-dev.json; client chat .striplayout / /striplayout edits playerstatusstrip-hudlayout.json.

Fixes

  • Fixed duplicate hotkey exception after return-to-menu → re-enter world (mod system no longer fails to start).
  • Fixed empty status strip until full client restart when HUD state was stale across sessions.

Compatibility

Game version: as declared in mod metadata.
Dependency: game: *

Install / Update Notes

  • Replace existing mod folder contents with this package (flat copy into Mods/playerstatusstrip/).
  • Confirm version 0.1.28 in the mod list; pair with SlowTox Visualized 1.1.4 if you use that provider.

If anything still fails after update, check client-main.log for Failed to start system PlayerStatusStrip or duplicate hotkey errors.
