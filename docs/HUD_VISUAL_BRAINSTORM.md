# Брейншторм: визуал интоксикации и SlowTox (HUD)

Документ фиксирует **согласованные выводы** обсуждения: насколько хватает обычной прогресс-полоски, как читать «бафы/дебафы» в SlowTox, и как это ложится на макет (в т.ч. Figma). Детали кода и конфига — в **[SLOWTOX_RESEARCH.md](./SLOWTOX_RESEARCH.md)**.

---

## 1. Задачи UX

- Игрок **понимает**, какой **эффект** даёт текущая интоксикация.
- Игрок **ориентируется**, сколько **ещё пить / не пить**, чтобы **сдвинуться** к другой зоне эффектов (в перспективе — с учётом очереди токсинов и детоксикантов).
- В бою состояние **быстро сканируется** (цвет, зоны, иконки, без простыни текста).

---

## 2. Достаточно ли «обычной» прогресс-полоски?

| Задача | Одна `GuiElementStatbar`-подобная полоска |
|--------|-------------------------------------------|
| Показать **уровень** `intoxication` как один скаляр | **Да** — длина, цвет, при необходимости число / tooltip. |
| Показать **несколько независимых шкал** (очереди токсинов, фазы) | **Нет** — нужны дополнительные элементы или второй ряд. |
| Сделать **бафы/дебафы** так же читаемы, как в списке эффектов | **Слабо** — полоска не заменяет **иконки + краткие метки** без перегрузки. |

**Вывод:** одна полоска — хороший **каркас для главной шкалы** (`intoxication`). Полный запрос UX («какой эффект сейчас» + «куда движемся») разумно решать **слоями**:

1. **Главная шкала** с **цветовыми зонами** (пороги из конфига SlowTox / серверный `SlowTox.json`).
2. **Ряд иконок или компактных меток** активных направлений (сила, реген, защита, штраф скорости, передоз и т.д.) — несколько могут гореть **одновременно**.
3. Опционально — **вторая линия** или компактный блок для **`slowtox:newToxins` / `digestingToxins` / `slowtox:detoxicants`** (прогноз «что накапливается»).

Подходы API см. **[VISUALIZATION_OPTIONS.md](./VISUALIZATION_OPTIONS.md)**.

## 3. Интересующий вариант макета: «HUD-бар с зонами эффектов»

Идея: на одной оси **отображаются сегменты** (зоны), соответствующие интервалам из конфига, плюс маркер текущего значения.

Уточнение по модели SlowTox:

- Эффекты завязаны не на отдельные именованные бафы, а на **`intoxication`** и на **колоколообразные кривые** в заданных диапазонах (`rangeBottom` … `rangeTop`), с пиком у середины; отдельно — **линейный штраф скорости** и **ветка передоза**.
- Трейт **`composed`** меняет спад «пользы» на высокой интоксикации (см. upstream `CalculateBenefit*`).

Для мокапов в Figma опираться на **дефолты `IntoxicationConfig`** в upstream или на экспорт реального **`SlowTox.json`** с сервера — цифры могут отличаться.

Краткая карта зон (дефолты, одна шкала `intoxication`):

| Группа | Смысл для игрока (в UI) | Примечание |
|--------|-------------------------|------------|
| «Польза» 0.0–0.6 (пик ~0.3) | Реген HP, при наличии — реген стабильности | Узкий «sweet spot» ближе к низу шкалы |
| «Польза» 0.2–1.0 (пик ~0.6) | Сила / копание, редукция входящего урона, шанс игнора мелкого урона | Пересекается с первой группой |
| Штраф скорости | С ~0.6 до ~3.0 нарастает штраф ходьбы | Не колокол, отдельная логика |
| Передоз | С порога **≥ 1.2** (дефолт) — урон ядом | Конкурирует с регеном из ветки `UpdateHealth` |

Подробные пороги и формулы — в **[SLOWTOX_RESEARCH.md](./SLOWTOX_RESEARCH.md)** и в коде `SlowToxBehavior` / патчах урона.

## 4. Может ли игрок получать несколько «баффов» сразу?

**Да.** В SlowTox это не взаимоисключающие слоты: при одном значении `intoxication` могут одновременно быть ненулевыми:

- бонусы по разным кривым (сила/копание, реген, защита от урона и т.д.);
- параллельно — **штраф скорости**, если значение в зоне роста штрафа;
- при сильной интоксикации — **урон передоза** вместо/в дополнение к сценарию с регеном (см. условия в `UpdateHealth`).

Для UI: **одна позиция маркера на шкале может подсвечивать несколько зон/иконок** (не «только одна роль»).

---

## 5. Практика для Figma / макетов

- Разметить **легенду зон** с подписями, понятными игроку (не обязательно копировать внутренние имена полей).
- Заложить **пересечения** зон (два пика на разных интервалах).
- Предусмотреть **состояние «много иконок»** (3–5) без слипания — размер под `RuntimeEnv.GUIScale` / тест в игре.
- Опционально: отдельный блок **очереди** (токсины / детокс) для ответа на вопрос «что будет, если ещё выпить».
