# SlowToxVisualized -> Player Status HUD migration notes

## What changed

- `SlowToxVisualized` no longer renders a standalone HUD block (jug/gear/number).
- It now registers a provider in `Player Status HUD` (`playerstatusstrip`) and publishes statuses there.
- Intoxication is represented as a regular neutral status with tooltip containing the current intoxication level.

## Dependency model

- `SlowToxVisualized` now requires:
  - `slowtox`
  - `playerstatusstrip`

## Runtime behavior

- All effect icons/tooltips are generated by `SlowToxVisualized`.
- Rendering, ordering, animation, hit-test and tooltip panel are handled by `Player Status HUD`.
- `Player Status HUD` mock icons (`mock_*`) are dev-only and must not replace SlowToxVisualized production status icons.

## Config impact

- `slowtoxvisualized-hudlayout.json` is now used as data-layout source for SlowTox calculations and tooltip values.
- Screen placement and strip rendering are configured in `playerstatusstrip-hudlayout.json`.
- Recommended production baseline:
  - `playerstatusstrip-dev.json`: `DevMode=false`, `UseMockStatuses=false`

## Validation checklist

1. Build and deploy both mods.
2. Verify both dependencies are active in the mod list.
3. Confirm intoxication appears as a neutral strip status and tooltip shows current value.
4. Confirm SlowTox buff/debuff statuses still appear with expected tooltips.
