# Исследование мода SlowTox (upstream)

## Источник и снимок кода

- **Репозиторий:** [https://codeberg.org/xyrvexus/SlowTox](https://codeberg.org/xyrvexus/SlowTox)
- **Локальная копия:** `SlowTox-upstream/` в корне проекта `SlowToxVisualized` (получена командой `git clone`).
- **Просмотренный коммит:** `3ac355aeb47a7cdc803e025244c0d36adf88dcba` (2026-04-21), сообщение: «5.0.0 upgrade to dotnet 10…» — в том числе снятие жёсткой «только сервер» специфики в `modinfo`, чтобы клиентские моды могли объявлять зависимость от SlowTox.

## Назначение

Мод меняет модель опьянения в Vintage Story: вместо мгновенного роста «интоксикации» из еды/напитков вводится **очередь токсинов**, усвоение, толерантность и **«польза» в определённом диапазоне опьянения** (сила, реген, снижение урона и т.д.), плюс штраф скорости ходьбы и урон при сильном опьянении.

В `modinfo.json`: версия **5.0.0**, тип `code`, зависимость только от `game: *`.

## Архитектура

| Компонент | Роль |
|-----------|------|
| `SlowToxSystem` (`ModSystem`) | Загрузка конфига, `Harmony.PatchAll()`, на **сервере** — подписка `OnEntitySpawn` для добавления поведения игрокам, регистрация чата `/tox`. |
| `SlowToxBehavior` (`EntityBehavior`, код `"slowtox"`) | Вся логика тиков, пересчёт `intoxication`, модификация статов, реген/урон. Выполняется только если `entity.World.Side == Server`. |
| Harmony-патчи | Перехват употребления еды/жидкости и отключение ванильных изменений `intoxication` в конкретных местах; опционально — совместимость с ACulinaryArtillery. |
| `assets/slowtox/config/traits.json` | Трейты персонажа, влияющие на статы `slowtox:*` и флаг `composed`. |
| Конфиг `SlowTox.json` | Сервер загружает с диска и кладёт в world config (base64); клиент читает из world config. |

**Замечание по клиенту:** в `SlowToxSystem` объявлено `public static ICoreClientAPI clientAPI`, но оно **нигде не присваивается** — задел под клиентские функции не подключён. Отдельный файл `StatusHud/StatusHudLoader.cs` **полностью закомментирован**: это был черновик загрузки опционального `StatusHud.dll` и регистрации элемента HUD — фактической интеграции нет.

## Модель данных опьянения

Игра по-прежнему использует ванильный атрибут сущности **`intoxication`** (float), но его изменение из еды контролируется патчами (см. ниже).

Дополнительные поля в **`WatchedAttributes`** (с префиксом `slowtox:`):

- `newToxins` — только что принятые токсины (ещё не в «переваривании»).
- `lastKnownNewToxins` — вспомогательное значение для переноса в digestion между длинными тиками.
- `digestingToxins` — токсины в процессе усвоения (отсюда растёт текущая интоксикация по модели мода).
- `detoxicants` — «детоксиканты», уменьшают активную интоксикацию при переваривании.

Статы (через `entity.Stats`, blend **FlatMultiply**):

- `slowtox:tolerance` — толерантность; итог `Tolerance = max(0.1, blended)`. Есть накачиваемый модификатор `habitualTolerance`.
- `slowtox:benefitMult` — множитель «пользы» из трейтов/конфига персонажа.
- `slowtox:overintoxicationDamageMult` — множитель урона при передозе.

В код поведения также пишутся модификаторы:

- `meleeWeaponsDamage` / `miningSpeedMul` с ключом `"intoxicated"` — бонус силы/добычи в заданном диапазоне `intoxication`.
- `walkspeed` с ключом `"intoxicated"` — отрицательное значение как штраф к скорости.

## Тики и формулы (кратко)

- Короткий тик (~3 с игрового времени с нормализацией через календарь): переваривание токсинов, обновление силы и скорости ходьбы, стабильность (если есть атрибут `temporalStability`).
- Длинный тик (~6 с): перенос `newToxins` → `digestingToxins`, обновление привычной толерантности, здоровье (реген или урон при «передозе»).
- Режим **`debug`** (команда `/tox debug`): останавливает метаболизм токсинов и перенос в digestion на этом игроке — для отладки.

«Польза» и штрафы используют общую идею **колоколообразной кривой** относительно середины диапазона (`peak`), с особым случаем трейта **`composed`**: при нём спад «пользы» при высокой интоксикации выше пика отключается (ветки `falloffDisabled` в `CalculateBenefit*`).

## Harmony-патчи

| Файл | Цель |
|------|------|
| `Patch_CollectibleObject` (`tryEatStop`) | После почти полного приёма еды (`secondsUsed >= 0.95`): если в nutrition интоксикация неотрицательна — добавить к `slowtox:newToxins`; если отрицательна (детокс в данных еды) — к `slowtox:detoxicants`. **Postfix принудительно восстанавливает `intoxication` на значение до приёма еды**, чтобы ваниль не меняла его мгновенно. |
| `Patch_BlockLiquidContainerBase` | То же для контейнеров с жидкостью; доза считается от доли выпитого объёма (до 1 л за глоток). |
| `Patch_EntityBehaviorHunger` (`detox`) | Сохраняет и **возвращает** прежнее `intoxication` после ванильного детокса — иначе ваниль ломала бы новую модель. |
| `Patch_WearableStats_HandleDamaged` | После расчёта урона по броне: при наличии `SlowToxBehavior` и подходящем источнике урона — снижение урона и шанс обнулить «мелкий» урон по кривой от опьянения. |
| `Patch_ACulinaryArtilllery_BlockBottle` | Условный патч (`Prepare`: загружена сборка `ACulinaryArtillery`): логика как у жидкостей, свои литры за глоток. Namespace в коде — `HydrateOrDiedrate.patches` (исторический артефакт имени). |

## Конфигурация (`IntoxicationConfig`)

Файл генерируется/хранится как **`SlowTox.json`**. Ключевые группы параметров:

- Скорость процессов: `IntoxicationSpeed`, `IntakeRate`, `DecayRate`, `HabitualToleranceChangeRate`.
- Порог передоза: `OverintoxicationThreshold`.
- Диапазоны и максимумы для регена здоровья, регена стабильности, бонуса к урону/копанию, редукции урона, штрафа скорости и т.д.

Строки для экрана настроек мода лежат в `assets/slowtox/lang/en.json` (и `ru.json`) с ключами вида `slowtox:Config.Setting.*`.

## Почему «ничего не видно» в игре

1. **Нет клиентского UI/HUD в активном коде:** закомментирован `StatusHudLoader`; нет `StartClientSide` с регистрацией элементов интерфейса, шейпов или оверлея.
2. **Обратная связь только косвенная:** изменения статов (скорость, урон) заметны по геймплею, но **нет отдельной полоски/иконок «уровня опьянения», толерантности, очереди токсинов** — игрок не видит внутренние `newToxins` / `digestingToxins` / `detoxicants`.
3. **Чат `/tox`** выводит числа только при включённом `CommandsAllowed` и включённой системе — это не постоянная визуализация.
4. Ванильный индикатор опьянения (если он есть в текущей версии VS) отражает в основном **`intoxication`**, но не разделяет стадии «в желудке» vs «в крови» и не показывает дизайнерски ваши бафы.

## Направления для проекта «SlowToxVisualized»

Ниже — не требования, а логичное продолжение исследования. Варианты API и ссылки на документацию см. **[VISUALIZATION_OPTIONS.md](./VISUALIZATION_OPTIONS.md)** (документ дополняется по мере работы).

- **HUD:** полоска или несколько сегментов: текущая `intoxication`, опционально очередь (`newToxins` + `digestingToxins`), индикатор детоксикантов; подписи/иконки из вашего дизайна.
- **Синхронизация:** убедиться, что нужные `WatchedAttributes` реплицируются на клиент (в VS это обычно так для синхронизируемых полей сущности игрока); при необходимости — пакеты или зеркалирование в клиентском поведении только для отображения.
- **Зависимость от SlowTox:** после изменения `modinfo` upstream-клиенты могут зависеть от `slowtox`; ваш мод может быть чисто клиентским оверлеем или форком с визуализацией.
- **Сборка upstream:** `SlowTox.csproj` таргетирует **net10.0**, ссылки на `VINTAGE_STORY` (API, Lib, Mods). Сборка через `build.ps1` / Cake по README при необходимости.
